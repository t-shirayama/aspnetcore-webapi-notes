# 例外ハンドリングミドルウェア

例外ハンドリングミドルウェアは、未処理例外をまとめて処理するための仕組みです。

```csharp
builder.Services.AddProblemDetails();

var app = builder.Build();

app.UseExceptionHandler();
```

個別の Controller や Service で毎回 `try-catch` を書くと、エラー形式がばらつきやすくなります。

共通の場所で例外を受け止めると、ログ出力とレスポンス形式を統一できます。

ただし、想定できる業務エラーまで何でも例外にする必要はありません。
