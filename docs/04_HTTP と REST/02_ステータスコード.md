# ステータスコード

ステータスコードは、リクエストの結果をクライアントに伝える番号です。

| コード | 意味 |
| --- | --- |
| `200 OK` | 成功 |
| `201 Created` | 作成成功 |
| `204 No Content` | 成功したが本文なし |
| `400 Bad Request` | リクエストが不正 |
| `401 Unauthorized` | 認証が必要 |
| `403 Forbidden` | 権限がない |
| `404 Not Found` | 見つからない |
| `409 Conflict` | 重複や状態競合がある |
| `422 Unprocessable Content` | 形式は正しいが意味的に処理できない |
| `500 Internal Server Error` | サーバーエラー |

Web API では、レスポンス本文だけでなく **適切なステータスコード** を返すことが重要です。

## 現場での判断基準

`401 Unauthorized` は、ログインしていない、トークンがない、トークンが無効のように **認証ができていない** 場合に使います。`403 Forbidden` は、認証済みだがその操作をする権限がない場合に使います。

`409 Conflict` は、メールアドレスの重複、楽観ロックの競合、公開済み記事を下書きに戻せないなど、現在の状態とリクエストが衝突している場合に候補になります。

`422 Unprocessable Content` は、JSON の形式や型は正しいが、意味的に処理できない入力に使うことがあります。ただし、入力エラーを `400 Bad Request` に寄せるチームも多いため、採用するかどうかはプロジェクトの方針として決めます。

`500 Internal Server Error` は基本的に想定外エラーです。重複、在庫不足、状態不正のような業務エラーに `500` を乱用しないようにします。
