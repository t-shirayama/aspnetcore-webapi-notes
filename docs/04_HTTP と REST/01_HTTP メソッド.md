# HTTP メソッド

HTTP メソッドは、リソースに対して何をしたいかを表します。

| メソッド | よく使う意味 |
| --- | --- |
| `GET` | 取得 |
| `POST` | 作成、処理の実行 |
| `PUT` | 全体更新 |
| `PATCH` | 部分更新 |
| `DELETE` | 削除 |

API 設計では、**動詞を URL に入れすぎず、HTTP メソッドで操作を表す** と整理しやすくなります。

```text
良い例: GET /articles
良い例: DELETE /articles/123

避けたい例: GET /getArticles
避けたい例: POST /deleteArticle
```

## 現場での判断基準

`GET` は基本的に副作用を持たせません。検索、詳細取得、一覧取得のように「読むだけ」の操作に使います。

`POST` は作成だけでなく、検索条件が大きい検索、再計算、承認、メール送信のような **処理の実行** にも使われます。ただし、何でも `POST` に寄せると API の意図が読みにくくなります。

`PUT` はリソース全体を置き換える更新、`PATCH` は一部だけを変える更新に使うと整理しやすいです。

`DELETE` は削除に成功し、返す本文がなければ `204 No Content` がよく使われます。

## よくある失敗

- `GET` で登録、削除、既読化などの状態変更をしてしまう
- URL に `getArticles`、`createArticle`、`deleteArticle` のような動詞を入れすぎる
- `PUT` と `PATCH` の違いを決めず、更新 API の仕様が曖昧になる
- 作成、更新、処理実行をすべて `POST` にして、クライアントから意味を判断しづらくなる
