# サーバーエラーの扱い

サーバーエラーは、クライアントがリクエストを直しても解決できない問題です。

代表的なものは `500 Internal Server Error` です。

```json
{
  "title": "An unexpected error occurred.",
  "status": 500
}
```

`500` を返すときに、例外メッセージやスタックトレースをそのまま返してはいけません。内部構造や秘密情報が外部に漏れる可能性があります。

外部には汎用的なエラーを返し、詳細はログに残します。

`500` は、基本的に想定外の失敗を表します。業務ルールで予測できる失敗は、`400`、`404`、`409` などのクライアントエラーとして設計する方が、クライアント側も次の行動を判断しやすくなります。

```text
良い例: 重複登録 -> 409 Conflict
良い例: 対象なし -> 404 Not Found
避けたい例: 業務エラーをすべて 500 Internal Server Error
```
