# 冪等性

冪等性は、同じ操作を 1 回実行しても複数回実行しても、結果が同じになる性質です。

HTTP API では、リトライや二重送信を考えるときに重要です。

| メソッド | 冪等性 | 考え方 |
| --- | --- | --- |
| `GET` | ある | 何度取得しても状態は変わらない |
| `PUT` | ある | 同じ内容で何度更新しても結果は同じ |
| `DELETE` | あると考えることが多い | 何度削除しても最終的に存在しない |
| `POST` | ないことが多い | 何度送ると複数作成される可能性がある |
| `PATCH` | 設計次第 | 差分の表し方によって変わる |

例えば、注文作成 API に同じ `POST` が 2 回届くと、注文が 2 件作られるかもしれません。

```text
POST /orders
POST /orders
```

このような二重作成を避けたい場合は、冪等キーを使う設計があります。

```http
Idempotency-Key: 8f7b...
```

冪等性は、HTTP メソッドの意味だけでなく、API の実装と業務要件で決まります。
