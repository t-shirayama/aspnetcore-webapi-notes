# HTTPS

HTTPS は、通信内容を暗号化して途中で盗み見や改ざんされにくくする仕組みです。

Web API では、トークン、Cookie、個人情報、入力データを扱うため、基本的に HTTPS を前提にします。

```csharp
app.UseHttpsRedirection();
```

開発環境では HTTP で動かすことがありますが、本番環境で平文 HTTP のまま認証情報を扱ってはいけません。

Cookie を使う場合は `Secure` 属性を付け、HTTPS でだけ送信されるようにします。

## よくある失敗

- ログイン API やトークン付き API を HTTP で公開する
- リバースプロキシ配下で HTTPS 判定やリダイレクト設定を確認していない
- 開発環境の例外的な HTTP 設定を本番にも持ち込む
