# 設定管理

設定管理では、環境ごとに変わる値をコードから分離します。

環境ごとに変わる値:

- DB 接続文字列
- 外部 API の URL
- ログレベル
- CORS の許可オリジン
- JWT の発行者や対象者
- Secret 名や Key Vault の参照

開発環境では `appsettings.Development.json` を使えますが、本番の秘密情報を JSON ファイルに直接置くべきではありません。

本番では、環境変数、Secret Manager、クラウドの Secret 管理サービスなどを使います。

設定管理で大切なのは、**同じアプリケーションを設定だけ変えて環境ごとに動かせること** です。
