# 論理削除

論理削除は、レコードを物理的に消さず、削除済みフラグなどで扱う方法です。

```csharp
public bool IsDeleted { get; set; }
```

監査や復元が必要な業務では論理削除が選ばれることがあります。

ただし、すべての検索で `IsDeleted == false` を忘れずに入れる必要があります。EF Core のグローバルクエリフィルターを使う方法もあります。

論理削除は便利ですが、データ量や一意制約、運用手順にも影響します。

例えばメールアドレスに一意制約がある場合、削除済みユーザーと同じメールアドレスで再登録できるかを決める必要があります。

管理画面や監査では、削除済みデータを見たい場合があります。一方、通常の API では削除済みデータが混ざると事故になります。

## よくある失敗

- 一覧では除外しているが、詳細取得では削除済みを返してしまう
- 論理削除済みデータが一意制約に残り、再登録できない
- 物理削除しない前提なのに、保持期間や監査の方針が決まっていない
