# 楽観ロック

楽観ロックは、同じデータを複数人が更新したときの競合を検出する仕組みです。

EF Core では、行バージョンや更新日時を使って競合を検出できます。

競合が起きた場合は、`409 Conflict` として返す設計が候補になります。

```text
ユーザー A が取得
ユーザー B が更新
ユーザー A が古い情報で更新
```

このような場面で、後勝ちで上書きしてよいのか、競合として止めるのかを決めます。

```csharp
public class Article
{
    public int Id { get; set; }
    public string Title { get; set; } = "";
    public byte[] RowVersion { get; set; } = [];
}
```

競合を検出したら、単に再保存するのではなく、クライアントへ競合を知らせます。業務 API では「最後に保存した人が勝つ」でよいかを必ず確認します。
