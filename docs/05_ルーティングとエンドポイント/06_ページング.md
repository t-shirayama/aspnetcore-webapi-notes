# ページング

ページングは、一覧 API でデータを少しずつ返すための設計です。

全件を一度に返すと、レスポンスが重くなり、DB やネットワークに負荷がかかります。

よくある形式:

```text
GET /articles?page=1&pageSize=20
GET /articles?offset=0&limit=20
```

レスポンスには、現在のページだけでなく、必要に応じて総件数や次ページの有無を含めます。

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "totalCount": 125
}
```

大量データやリアルタイム性が高い一覧では、カーソルページングも候補になります。

```text
GET /articles?cursor=eyJpZCI6MTAwfQ&limit=20
```

ページングは、API 利用者の画面表示と DB の負荷の両方に影響します。
