# エラー形式

API のエラー形式は、プロジェクト内で統一します。

エラーごとに形式が違うと、フロントエンドや API 利用者が扱いにくくなります。

ASP.NET Core では Problem Details を使うと、標準的な形に寄せられます。

```json
{
  "type": "https://example.com/errors/validation",
  "title": "Validation failed",
  "status": 400,
  "detail": "One or more validation errors occurred.",
  "traceId": "00-..."
}
```

バリデーションエラーでは、項目ごとのエラーを返す必要があります。

```json
{
  "title": "Validation failed",
  "status": 400,
  "errors": {
    "title": ["The Title field is required."]
  }
}
```

エラー形式では、クライアントが次に何をすればよいか分かることが重要です。
