# 日付と時刻の扱い

API と DB で日付・時刻を扱うときは、タイムゾーンを意識します。

実務では、保存は UTC、表示はユーザーのタイムゾーンに変換する方針がよく使われます。

```csharp
var now = DateTimeOffset.UtcNow;
```

`DateTime`、`DateTimeOffset`、日付だけを表す `DateOnly` のどれを使うかも設計対象です。

期限、予約、集計期間などは、境界値でバグが出やすい領域です。

| 型 | 向いている用途 |
| --- | --- |
| `DateTimeOffset` | API や DB に保存する日時 |
| `DateTime` | 既存ライブラリや内部処理で必要な場合 |
| `DateOnly` | 誕生日、営業日、締切日など日付だけ |

API では、タイムゾーンを含む値なのか、日付だけなのかを DTO で明確にします。
