# リトライ

リトライは、一時的な失敗に対して同じ処理をもう一度試すことです。

ネットワークの瞬断や一時的な外部 API の失敗には有効です。一方で、何でもリトライすると障害を悪化させることがあります。

```text
有効な例: 一時的な 503、ネットワークの瞬断
注意が必要: POST、決済、メール送信、在庫更新
```

副作用のある処理をリトライする場合は、冪等性を考えます。同じリクエストを複数回送っても二重作成や二重送信にならない設計が必要です。

リトライは、回数、間隔、対象エラーを決めて使います。無限リトライは避けます。

## よくある失敗

- すべてのエラーを無条件にリトライする
- `POST` のリトライで二重登録が起きる
- リトライで外部サービスへの負荷をさらに増やす
